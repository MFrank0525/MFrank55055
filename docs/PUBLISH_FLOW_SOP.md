# Publish Flow SOP

适用入口：

```powershell
npm run business:publish -- --job <publish-from-spu.job.json>
```

这份 SOP 是发布业务的最高优先级执行规范。目标是让 AI 助手按同一套稳定思路操作浏览器，不再靠猜测、模糊匹配或错误兜底继续流程。

## Core Principles

1. 先确认当前页面和当前模块，再操作字段。
2. 先找字段标签，再找标签右侧同一行控件。
3. 每一步都必须“操作 + 同控件读回确认”。
4. 一个模块未通过，不进入下一个模块。
5. 不自创信息，不扩展别名，不猜字段含义。
6. 不允许触碰规则之外的字段、图片区域、规格值或类目属性。
7. 一旦页面身份错误、字段错误、值错误、出现红字报错，立即停下修复，不带错继续。
8. 浏览器默认单页执行，禁止在同一条业务链路里不断新开多个页面。

## Stable Locator Strategy

浏览器定位优先级固定如下：

1. 当前页签 / 模块锚点
2. 当前字段标签
3. 标签右侧同一行、同容器控件
4. 同控件读回确认
5. 坐标仅作为最后兜底

禁止事项：

- 禁止全页乱扫相似控件
- 禁止用错误别名或 OCR 猜词继续执行
- 禁止把“点了”当成“成功了”
- 禁止为了局部验证不断新开多个页签

## Browser Page Discipline

浏览器执行统一遵守下面规则：

1. 单一业务链路默认只保留一个主页面。
2. 必须新开页面时，只允许为了平台自然跳转或新页面承接结果。
3. 一旦目标页面已经打开，应主动关闭多余旧页签。
4. 不允许通过不断新开查询页、发布页、调试页来推动流程。
5. 若页面错乱，优先复用当前主页面回到标准入口，而不是继续叠加新页面。

目标：

- 降低平台风控概率
- 降低会话混乱概率
- 保持页面身份、登录态、操作上下文稳定

## Module Order

严格按下面顺序执行：

1. 标品检索
2. 基础信息
3. 图文信息
4. 价格库存
5. 服务与履约
6. 发布前检查 / 发布商品

规则：

- 当前模块未验证通过，不进入下一模块
- 任一模块失败，立即停下修复
- 修复成功后，继续当前模块；必要时刷新发布页后从基础信息按顺序重跑

## 最新入口（2026-03 已验证）

标准起点不是旧草稿页，也不是手动到处点菜单找入口，而是先进入平台标品页：

- 页面定位：`商品发布 -> 标品管理/标品检索 -> 平台标品`
- 稳定 URL：`https://fxg.jinritemai.com/ffa/g/spu-record?type=create&btm_ppre=a2427.b76571.c902327.d871297&btm_pre=a2427.b39372.c67909.d0&btm_show_id=1f4fb4cd-7a30-4c1d-8d9c-6250a9e7a466`
- 后续入口：在查询结果行点击 `发布商品`，进入 `/ffa/g/create` 发布页

执行要求：

1. 先确认自己在 `平台标品` 查询页，再填品牌/SPU。
2. 只有结果行里的品牌和 SPU 与表格完全一致，才允许点 `发布商品`。
3. 一旦已经进入 `/ffa/g/create` 发布页，后续模块都在这个发布页里完成，不重新乱开页面。

## 1. 标品检索

输入依据：

- 品牌：只能使用表格里的品牌信息
- SPU：只能使用表格里的 SPU 信息

允许操作：

- 填品牌
- 填 SPU
- 自检输入框读回
- 点击查询

禁止操作：

- 不允许自创品牌或 SPU
- 不允许推测替换值
- 不允许输入不一致时继续查询

成功判定：

- 品牌输入框读回值与表格一致
- SPU 输入框读回值与表格一致
- 查询结果强匹配表格品牌 + 表格 SPU

失败回退：

- 输入读回不一致：重新定位同一输入框再填
- 查询结果不匹配：停止，不进入发布页

## 2. 基础信息

输入依据：

- 标题：表格
- 短标题：表格
- 型号规格：固定值 `盒装`

前置硬校验：

- 从类目属性模块读取 `医疗器械备案/注册号`
- 与表格里的 SPU 严格比对

规则：

- 只有 `医疗器械备案/注册号 == 表格 SPU` 才允许继续
- 类目属性模块里只允许填写 `型号规格 = 盒装`
- 其余类目属性任何字段都不能改

失败回退：

- 备案号不一致：立即回到标品检索重查
- 误触其他类目属性：刷新页面，从基础信息重做

## 3. 图文信息

### 主图

规则：

- 只在 `主图` 模块上传
- 主图首位只用商品文件夹里的 `主图01`
- 其余辅助图固定使用 `input/fixed-main-images` 下的 4 张固定图片
- 不再调取商品文件夹里的辅助图

### 主图 3:4

规则：

- 主图上传完成后，在 `主图3:4` 模块点击 `从1:1主图智能裁剪`
- 不手动往 `主图3:4` 上传图片

### 白底图

规则：

- 不主动上传
- 如果平台自动填充，不干预

### 商品详情

规则：

1. 先点击按钮 `从主图填入`
2. 确认这一步已生效
3. 再额外上传指定图片

额外允许上传的图片名只包含：

- `医疗器械注册证`
- `白装展开图`
- `包装展开图`

成功判定：

- 主图区域顺序正确
- `主图3:4` 已按规则生成
- 商品详情结果 = 主图填入内容 + 指定额外图片

## 4. 价格库存

输入依据：

- 价格：固定价格数组
- 库存：固定库存值

规则：

- 只允许填写 `价格与库存` 模块里的价格框和库存框
- 商品规格子模块里的规格名和规格值不允许编辑
- 商品规格完全依赖规格模板
- 不允许把价格或库存写进规格值框

如果商品规格模块报错或出现红字提醒：

- 不去手动编辑规格名或规格值
- 直接刷新当前发布页
- 从基础信息开始按顺序重新执行

成功判定：

- 每一行价格和库存都从同一行读回正确
- 规格值框未被价格/库存污染

## 5. 服务与履约

规则：

- 只认页面真实字段 `运费模板`
- 不允许使用错误别名
- 必须在 `服务与履约` 页签下操作
- 只操作 `运费模板` 标签右侧同一行的下拉框
- 选择名称包含 `延草运费` 的模板

成功判定：

- 从同一个下拉框读回值，结果包含 `延草运费`

其他固定项：

- 发货模式：现货发货模式
- 发货时间：48 小时
- 商品状态：上架

## 6. 发布前检查

规则：

- 只有前面所有模块都通过，才允许进入发布前最后一步
- 如果有任何模块仍未通过，不允许点击 `发布商品`

成功判定：

- 标品正确
- 备案号校验正确
- 基础信息正确
- 图文信息正确
- 价格库存正确
- 服务与履约正确

## Failure Handling

统一失败处理原则：

1. 先判断失败属于哪个模块
2. 只在该模块范围内修复
3. 修不好，再刷新发布页并按顺序重跑
4. 不允许跳过失败模块继续后续操作

## AI Assistant Rules

AI 助手必须记住：

- 页面真实结构优先于历史经验
- 同控件闭环优先于“点了就算成功”
- 模块顺序优先于局部捷径
- 精确字段名优先于模糊匹配
- 语义定位优先于坐标定位
- 失败即停优先于带错继续

如果未来新增业务细则，应优先更新这份 SOP，再执行新的自动化规则。
