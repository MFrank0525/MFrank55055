# 步骤 04：产品图片文件夹整理

## 目标

把 Dreamina CLI 生成并打好店铺水印的图片，整理成正式产品图片文件夹。

## 动作脚本

- [jimeng-assets.ts](C:/Users/ffc/.openclaw/workspace/douyin-title-generator/src/autolist/jimeng-assets.ts)

## 输入

- 5 轮 Dreamina CLI 生图结果
- 5 轮已打店铺水印的图片
- 店铺目录根路径

## 输出

- 店铺根目录中的正式产品图片文件夹
- 每个产品图片文件夹中的首位主图文件

## 操作说明

### 水印顺序

```text
01 延草纲目理疗器械旗舰店
02 延草纲目健康护理专营店
03 延草纲目个护保健专营店
04 延草纲目康复理疗专营店
05 延草纲目医疗保健专营店
```

### 图片命名规则

```text
带有品牌的产品通用名称 + 水印内容 + 编号
```

### 产品图片文件夹命名规则

```text
带有品牌的产品通用名称 + 水印 + 编号01到N
```

## 执行要求

1. 各轮生图结果先进入任务临时区，不直接写正式店铺目录。
2. 只有 5 个批次全部完成后，才统一写正式店铺目录。
3. 每一批图片先进入对应店铺文件夹，不跨店铺分发。
4. 店铺与水印内容必须严格一一对应，只允许使用上面 5 个固定水印名称，不允许自行添加、减少、改名或调换顺序。
5. 产品图片文件夹必须建在对应店铺目录内。
6. 对应水印图移入各自产品图片文件夹时，必须用剪切粘贴方式移入。
7. 产品图片文件夹外不允许残留同批次图片，避免干扰后续上架流程。
8. Dreamina CLI 生成并打了店铺水印的图片，就是后续上架商品时放在首位的主图。

## 失败条件

- 图片落到错误店铺
- 水印内容与店铺不对应
- 产品图片文件夹命名不符合规则
- 产品图片文件夹外仍残留同批次图片

## 人工检查点

1. 检查每个店铺目录内的产品图片文件夹命名是否正确。
2. 检查产品图片文件夹外是否还有同批次图片残留。
3. 检查主图是否为打好店铺水印的 Dreamina 图片。
4. 检查 5 个店铺对应的水印内容是否严格为：理疗器械旗舰店、健康护理专营店、个护保健专营店、康复理疗专营店、医疗保健专营店。
