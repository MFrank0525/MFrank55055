# 步骤 01：豆包卖点生成

## 目标

根据当前白底图生成结构化卖点，并产出后续链路必须使用的核心字段：

- `userCognitionName`
- `brandedGenericName`
- `brand`
- `sellingPointText`

## 动作脚本

- [doubao-selling-points.ts](C:/Users/ffc/.openclaw/workspace/douyin-title-generator/src/autolist/doubao-selling-points.ts)
- [submit.ts](C:/Users/ffc/.openclaw/workspace/douyin-title-generator/src/doubao/submit.ts)
- [capture.ts](C:/Users/ffc/.openclaw/workspace/douyin-title-generator/src/doubao/capture.ts)

## 输入

- 当前待处理白底图
- 固定豆包对话
- 固定卖点指令

## 输出

- `sellingPointText`
- `userCognitionName`
- `brandedGenericName`
- `brand`
- 豆包原始结果留档

## 操作说明

### 固定对话标题

```text
产品卖点生成
```

### 豆包卖点指令

```text
产品卖点生成规则（完整整理版严格执行版）

一、产品识别与命名规则
基础信息提取
核心品牌：包装最醒目品牌名称
无品牌通用原名：包装官方标注全称
包装凸显部位：包装主视觉强调的身体部位
剂型判定优先级：实物容器形态＞包装印刷剂型＞质地描述
喷瓶 / 泵头＝喷剂
软管＝乳 / 软膏
罐装＝乳霜
贴片＝膏贴

用户认知产品名推导（固定结构，不可更改）
通用名含部位：医用 + 部位 + 剂型
通用名无部位：医用 + 包装部位 + 剂型
注意：用户认知产品名不含品牌

身份资质判定
械字号：医疗器械
药字号：药品
食字号：蓝帽保健食品
OTC：非处方药

带品牌产品通用名：品牌 + 包装官方标题
存档规则：单品独立建档，同品永久复用，新品不混用旧品信息

二、8 个卖点固定顺序与内容（不可调换、不可遗漏）
01 品牌 + 用户认知产品名
02 带品牌的产品通用名称
03 资质对应固定文案
医疗器械：医疗器械认证，成分科学严谨，不添加科技狠活
蓝帽保健食品：蓝帽保健食品认证，成分科学严谨，不添加科技狠活
OTC 药品：OTC 药品认证，成分科学严谨，不添加科技狠活
04 固定文案：官方正品，正品保证，匠心甄选
05 核心成分
三源验证：注册批文备案信息 + 产品包装标注 + 抖音商城同款 / 相似品商品属性
械字号结合批文作用原理推导，仅提炼真实成分，禁止编造
产品通用名称未披露核心成分时，只选最核心一个即可
禁止将作用原理当作成分
06 规格含量，格式如：30g / 瓶
07 固定结构：为 XXXX 问题研发，为 XXXX 人群研发
信息来源：产品包装 + 抖音以图搜款同款 / 相似品详情
禁止编造，禁止极限、治愈类词汇
08 固定文案：图示产品使用步骤，突出产品的使用部位，不展示批准文号信息

三、禁用词汇与宣传规范
禁止使用夸张、极限、治愈、根治、最佳、第一、最强等违规宣传词
禁止出现：抖音、商城、热销、买送、炎症、草本、草药、治病、治疗及同类词汇

四、输出格式要求
仅输出一个段落，无换行、无编号、无解释
多个卖点词组用逗号隔开，中间无空格
严格按照 8 个卖点顺序输出
```

## 执行要求

1. 不新开卖点对话，只复用指定对话。
2. 当前不在指定对话时，先回到指定对话，再发送白底图和卖点指令。
3. 输出必须是单段文本，不换行，不解释，不编号。
4. 必须严格输出 8 个卖点，顺序固定。
5. 第一个卖点是“品牌 + 用户认知产品名”。
6. 用户认知产品名本身不含品牌，最终输出时再按规则拼成“品牌 + 用户认知产品名”。
7. 第二个卖点是“带品牌的产品通用名称”。
8. 名称要单品独立存档，同品永久复用，新品不混用旧品信息。
9. 每次都重新基于当前白底图和当前规则生成卖点，不做历史固化复用。
10. 抓取时只允许提取当前这一次的最新回答块，不允许混入历史对话、旧规则或旧答案。

## 失败条件

- 卖点无法提取
- 无法稳定拆出用户认知名和带品牌通用名称
- 输出格式不符合 8 段要求
- 豆包页面未进入指定对话

## 人工检查点

1. 检查卖点是否只有一个段落。
2. 检查是否正好 8 段。
3. 检查第一段是否为“品牌 + 用户认知产品名”。
4. 检查第二段是否为带品牌的产品通用名称。
