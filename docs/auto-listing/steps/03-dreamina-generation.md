# 步骤 03：生图与水印

## 目标

读取 5 份 Word 文档，逐份调用配置好的生图 provider，并为后续本地水印与产品文件夹整理准备原始图片。

## 动作脚本

- [jimeng-assets.ts](src/autolist/jimeng-assets.ts)
- [local-watermark.ts](src/autolist/local-watermark.ts)
- [local-watermark.py](src/autolist/local-watermark.py)

## 输入

- 5 份 Word 文档
- 生图 provider 配置
- 生图模型参数

## 输出

- 每轮生图任务的 `submit_id`
- 每轮实际送给生图模型的 `dreamina-prompt.txt`
- 每轮生图结果或失败原因

## 操作说明

### 即梦指令1模板

```text
【产品海报设计】将参照的白底图里的产品主体（主体可能是外包装盒和内容物的组合形式，也可能是一个单一的主体）融入到海报设计图中，要求白底图里的产品外观和产品主体不能随着海报设计改变和减少（指的是如果是外包装盒和内容物的组合，那就都要展示在海报图中，不得减少只保留某个部分），尽量保持产品外观清晰可见上面的字体，以免消费者看不清楚产品，但是产品主体的尺寸可以调整以更适应整张海报设计图，并且为产品主体设置合适的光影轮廓效果，使之看起来与整体的海报设计更加的契合，C4D风格，OC渲染，电商海报风格，有艺术设计感的主标题：{{带有品牌的用户认知名}}，有艺术设计感的副标题：{{带有品牌的产品通用名称}}。
```

### 水印内容顺序

```text
01 延草纲目理疗器械旗舰店
02 延草纲目健康护理专营店
03 延草纲目个护保健专营店
04 延草纲目康复理疗专营店
05 延草纲目医疗保健专营店
```

## 执行要求

1. 每次只处理一份 Word 文档。
2. 生图模型实际消费的提示词只取：
   - 第一段：海报设计指令
   - 最后一段：当前轮 DeepSeek 提示词
3. 中间豆包卖点段不直接送给生图模型。
4. 文案内容必须严格按照 Word 文档中的可采纳内容执行，不允许自行添加，也不允许自行减少。
5. 水印内容必须严格只按上面的 5 个名称和顺序执行，不允许自行添加、减少、改名或调换顺序。
6. 每轮任务都必须单独记录真实提交文案。
7. `query_result` 一旦返回 `gen_status=fail`，立刻失败，不继续等待。
8. 正式生图前先执行一次 provider 配置预检；若 API Key、模型、接口地址缺失，立刻停止，不进入生图提交。
9. 任何失败都不允许提前写正式店铺目录。
10. 当前 Mac 飞书真实流程默认使用 OpenAI-compatible 中转站模型 `gpt-image-2`，默认要求每轮最终累计得到 4 张图。
11. 如果 provider 返回余额、额度、计费不足等错误，必须明确提示充值中转站账号。
12. 如果流程曾在生图或后续步骤中断，已成功生成并保存的图片必须优先复用，只补缺失图片，不允许无必要重复消耗额度重新生成。
13. 如果前面的步骤已经完成且没有问题，修复问题后必须从当前出错步骤继续，不重复执行已完成步骤。

## 失败条件

- 生图 provider 提交失败
- 生图 provider 余额不足、额度不足、API Key 无效或账户不可用
- `query_result` 返回 `fail`
- 生图完成但无图片文件
- Word 文档读取失败

## 人工检查点

1. 核对 `dreamina-prompt.txt` 是否只包含第一段指令和当前轮 DeepSeek 提示词。
2. 核对第一段指令是否为最新固定版本。
3. 查看失败时的 `fail_reason` 是否清晰。
4. 确认没有把豆包长卖点段直接送给生图模型。
5. 确认水印内容顺序严格为：理疗器械旗舰店、健康护理专营店、个护保健专营店、康复理疗专营店、医疗保健专营店。
