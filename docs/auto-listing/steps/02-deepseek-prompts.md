# 步骤 02：DeepSeek 海报提示词

## 目标

基于飞书卖点生成海报提示词，并按产品类目保存成对应数量的 Word 文档。

## 动作脚本

- [deepseek-prompts.ts](src/autolist/deepseek-prompts.ts)
- [deepseek-word-docs.ts](src/autolist/deepseek-word-docs.ts)
- [docx-lite.ts](src/autolist/docx-lite.ts)

## 输入

- `sellingPointText`
- 固定 DeepSeek 对话
- 指令1
- 指令2

## 输出

- 医疗器械 / 保健食品：5 段 DeepSeek 提示词，5 份 Word 文档：
  - `主图提示词01.docx`
  - `主图提示词02.docx`
  - `主图提示词03.docx`
  - `主图提示词04.docx`
  - `主图提示词05.docx`
- 非处方药：只使用前 3 段 DeepSeek 提示词，落地 3 份 Word 文档：
  - `主图提示词01.docx`
  - `主图提示词02.docx`
  - `主图提示词03.docx`

## 操作说明

### 固定对话标题

```text
日式医用贴膏海报设计
```

### 指令1

```text
主题海报视觉设计
```

### 指令2

```text
海报视觉设计生成规则
1. 输出格式
每个海报描述为由逗号隔开的关键词段落，不写长句，不繁琐。
2. 数量要求
每次需设计5款不同的电商海报。
3. 场景关联
海报场景必须与产品卖点强相关（如：持续发热渗透、远红外陶瓷粉、缓解疼痛、不添加科技狠活、正品保证等），不再限定行业。
4. 内容细节
场景元素需丰富、具象、细腻：包括背景、道具、光线层次（高光、阴影、透射、反射）、材质纹理（木纹、金属、布料、水雾）、微观粒子轨迹、使用步骤拆解特写等。
必须融入产品卖点：成分科学严谨、不添加科技狠活、官方正品、正品保证、匠心甄选等，通过视觉元素（印章、标签、光效、防伪标等）体现。
图示产品使用步骤，突出产品使用部位（如膝盖）。
增强消费者信任度与转化率：展示正品防伪元素、科学标签、安全承诺等。
5. 禁止展示内容
不展示医疗器械备案注册号。
不展示保健食品注册批号。
不展示药品国药准字批号。
不展示具体人物形象，仅可展示使用部位。
不表明医生身份。
不展示人体、骨骼、穴位、疼痛部位、治疗过程、功效渗透、能量进入身体、疾病对比或医疗效果示意。
6. 文字风格
文案字体需有艺术设计感（如书法体、科技体、雕刻体等）。
部分要求使用繁体字时，必须按指令执行。
7. 创新性
当要求不同于之前时，需避免重复已描述的场景风格。
```

### 重试指令

```text
必须直接输出恰好5段关键词提示词，每段单独换行，段内关键词之间用逗号隔开，不要解释，不要分析，不要标题。
```

## 执行要求

1. 不新开 DeepSeek 对话，只复用指定对话。
2. 发送文案必须严格是一个整段：
   - 指令1
   - 飞书 `产品卖点` 字段原文
   - 指令2
3. 不允许追加任何无关说明，也不允许在飞书 `产品卖点` 字段前自动拼接 `用户认知名`、`品牌+产品通用名称`、SPU 或其他结构化字段。
4. DeepSeek 默认返回 5 段关键词提示词；非处方药只取前 3 段落地 Word 文档。
5. 每份 Word 文档内容固定为三段：
   - 第一段：主图指令模板
   - 第二段：飞书 `产品卖点` 字段原文
   - 第三段：对应轮次的 DeepSeek 提示词
6. 如果第一次返回不足 5 段，只允许按固定重试指令补发 1 次，避免重复消耗网页账号额度。
7. 运行产物只允许保存本轮 prompt 之后的回复片段、提取后的关键词段和当前视口截图；不允许把整页历史对话或全页截图作为常规产物落盘。
8. 暂停信号只在步骤边界生效；DeepSeek 已提交后必须等本次返回或失败后才能暂停。

## 失败条件

- 返回提示词不足当前类目要求数量
- DeepSeek 未进入指定对话
- Word 文档落地数量不等于 5
- 运行产物包含整页历史对话正文或全页历史截图

## 人工检查点

1. 打开任意一份 Word，确认是三段结构。
2. 确认第三段确实是当前刚生成的 DeepSeek 提示词。
3. 确认当前类目要求数量的提示词符合“逗号关键词段落”格式。
