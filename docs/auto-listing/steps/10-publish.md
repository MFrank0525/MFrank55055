# 步骤 10：商品上架

## 目标

按店铺顺序逐个产品图片文件夹执行上架，并保证整个发布流程严格按表格和固定规则执行。

## 动作脚本

- [publish.ts](src/autolist/publish.ts)
- [publish-from-spu.ts](src/business/publish-from-spu.ts)
- 规则与动作分层说明：[publish-rule-action-separation.md](docs/auto-listing/publish-rule-action-separation.md)

## 输入

- 已完成商品信息回填的产品图片文件夹
- 已完成资质图补充的产品图片文件夹
- 店铺目录

## 输出

- 每个产品的发布结果

## 操作说明

### 发布执行原则

```text
一定要按照表格里的信息执行，包括标题、导购短标题、品牌信息、SPU信息等，不要盲目去猜测，按照表格里面的信息执行即可。
```

### 主图与详情图规则

```text
上架商品时，主图生成节点产出的并且打了店铺水印的图片就是要上传的放在首位的主图，产品图片文件夹里的资质图片就是上架商品时详情页指定要上传的详情页图片。
```

### 医疗器械注册证规则

```text
医疗器械类目产品上架时，进入最后的“其他信息”模块检查“医疗器械注册证”字段：如果该字段已有填充内容，不做任何修改，继续后续上架；如果该字段为空，则直接上传飞书表格里的第一张资质图片，不走下拉框选择 SPU 信息方案。上传后必须确认该字段计数大于 0，仍为空则停止并保留断点。
```

### 服务与履约完成规则

```text
服务与履约相关必填项不得只按点击动作是否成功判定，必须以页面最终读回状态为准。发布前必须确认：发货模式=现货、现货发货时间=48小时、运费模板已选择有效的延草运费模板、商品状态=上架。任一字段未读回为完成时，停止并保留断点，不允许继续发布。
```

### 店铺切换规则

```text
一个店铺的产品上架完成后，必须切换到下一个店铺，再执行下一个店铺的上架。
```

## 执行要求

1. 必须按店铺顺序执行，先完成当前店铺全部商品，再切换到下一个店铺。
2. 进入上架前，必须先检查右上角当前店铺是否与产品文件夹所在店铺一致。
3. 店铺不一致时，必须先走“切换组织/店铺”流程，切换成功后再继续。
4. 如果标品页仍在 loading、右上角店铺入口尚未渲染，不允许立刻判定店铺菜单缺失；必须等待店铺入口出现，必要时重进标品页后重试切店，最后一次仍失败才停止并截图。
5. 发布页“基础信息”模块是否激活，不能只依赖旧版字段；看到商品标题、导购短标题、商品类目、类目属性、品牌、医疗器械备案/注册号等当前页面字段时，也应判定基础信息模块可用。
6. 发布页因网络抖动出现内容消失、模块标签缺失、长时间 loading、系统繁忙或页面文本异常变短时，不允许继续点击和填表；必须先等待当前 `/ffa/g/create` 页恢复，必要时 reload 或重新进入当前 create URL，确认基础信息/图文信息/价格库存/服务与履约等模块和发布动作重新出现后再继续。
7. 发布页出现“数据异常请刷新重试”“网络异常”“系统繁忙”“请稍后重试”等可恢复平台异常时，即使页面上仍有模块标题或发布按钮，也不得判定页面可用；必须先刷新/重进 create URL，恢复后再继续动作。
8. 模块切换不能只依赖当前视窗内是否能看到字段；如果顶部 active tab 已经是目标模块，且主表单 DOM 中存在该模块字段，应视为模块已切换成功，再由后续字段级读回继续校验。
9. 必须严格按照标题表格中的标题、导购短标题、品牌、SPU信息执行。
10. 商品详情图必须先点击 `从主图填入`，再上传飞书资质图；同一个商品的飞书资质图只允许上传一批，禁止因为页面计数延迟而整批重复上传。
11. 商品详情图最终数量必须刚好等于 `从主图填入后的数量 + 飞书资质图数量`；少于或多于预期都必须停止并保留断点。
12. 商品详情图是否完成必须以最终读回计数为准；如果上传动作的即时状态未确认，但最终计数已经刚好等于预期数量，视为详情图完成，不允许因瞬时状态误判失败。
13. 白底图/3:4 主图禁用槽位的最终读回只允许统计图文信息主表单里的预览图，必须排除右下角 AI 助手、反馈按钮、填写助手示例等固定悬浮层或辅助说明区图片。
14. 不允许程序自行猜测、替换或模糊选相近标品。
15. 主图生成节点产出的水印图作为首位主图。
16. 产品图片文件夹中的资质图作为详情图。
17. 同一商品流程只允许保留一个活跃 `/ffa/g/create` 发布页；重开发布页前必须关闭旧发布页。
18. 发布前检查未通过、白底图/3:4 主图未清空、详情页未完成时，必须停止并记录 checkpoint，不允许点击 `发布商品`。
19. 断点重跑时，如果当前产品已有发布成功截图或发布结果，必须跳过已发布商品，避免重复提交。
20. 浏览器动作和业务规则必须分开保存：页面点击、上传、截图属于动作层；是否算发布成功、是否能继续、是否必须停止属于规则层。
21. 医疗器械类目的“医疗器械注册证”字段必须按规则处理：已有填充物不覆盖，空白时上传飞书第一张资质图并校验成功。
22. 服务与履约模块必须按最终读回状态确认完成，不能把“点击过某个选项”当成字段完成。

## 失败条件

- 店铺切换错误
- 标品查询值与表格不一致
- 主图或详情图识别错误
- 医疗器械类目“医疗器械注册证”字段为空且第一张飞书资质图上传失败
- 服务与履约字段最终读回缺失：发货模式、现货发货时间、运费模板、商品状态任一未完成
- 发布页关键字段未通过检查
- 发布页内容消失或模块标签缺失时未先恢复页面就继续点击/填表
- 同一商品存在多个活跃发布页
- 已发布商品被重复提交

## 人工检查点

1. 检查右上角当前店铺是否与产品所在店铺一致。
2. 检查发布页填入的标题、导购短标题、品牌、SPU是否与表格一致。
3. 检查主图是否为主图生成节点产出的水印图，详情图是否为资质图。
4. 检查白底图和主图 3:4 是否为空。
5. 检查当前商品是否已有发布成功记录，避免重复点击发布。
6. 医疗器械类目检查“其他信息/医疗器械注册证”是否已有内容；空白时应已上传飞书第一张资质图。
