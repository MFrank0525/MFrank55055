# 步骤 10：商品上架

## 目标

按店铺顺序逐个产品图片文件夹执行上架，并保证整个发布流程严格按表格和固定规则执行。

## 动作脚本

- [publish.ts](src/autolist/publish.ts)
- [publish-from-spu.ts](src/business/publish-from-spu.ts)
- 规则与动作分层说明：[publish-rule-action-separation.md](docs/auto-listing/publish-rule-action-separation.md)

## 输入

- 已完成商品信息回填的产品图片文件夹
- 已完成资质图补充的产品图片文件夹
- 店铺目录

## 输出

- 每个产品的发布结果

## 操作说明

### 发布执行原则

```text
一定要按照表格里的信息执行，包括标题、导购短标题、品牌信息、SPU信息等，不要盲目去猜测，按照表格里面的信息执行即可。
```

### 标品检索规则

```text
标品检索时，品牌必须严格使用表格里获取的品牌信息，SPU必须严格使用表格里获取的SPU信息，不允许程序自创或推测替换查询值。
动作层必须先在平台标品查询页填写并确认品牌，再填写并确认SPU，两个字段都自检通过后才允许点击“查询”；不得只填写SPU，不得在品牌未确认时依赖结果行匹配兜底。
```

### 发布模块顺序规则

```text
严格按照模块顺序执行：先完善基础信息，再完善图文信息，再完善价格库存，再完善服务与履约，最后才允许点击“发布商品”；任何一个模块失败或报错都必须先停止、检查、修复，修复成功后才能继续下一步。
```

### 保健食品发布模块顺序规则

```text
保健食品分支顺序如下。
保健食品分支必须按以下顺序执行：标品检索 -> 基础信息 -> 食品安全 -> 类目属性 -> 图文信息 -> 现货发货与48小时发货时间 -> 规格模板 -> 手动规格 -> 飞书规格替换 -> 价格库存 -> 服务与履约设置和读回 -> 包装标签图 -> 发布前检查 -> 最终发布。
医疗器械注册证逻辑不得对保健食品运行；保健食品食品安全、类目属性、包装标签图和规格替换动作不得对医疗器械运行。保健食品基础信息不要求也不填写 `型号规格=盒装`。
```

### 类目属性填写规则

```text
类目属性模块里只允许填写“型号规格=盒装”，其余任何字段都不能改动；一旦触碰到其他类目属性，必须立即刷新页面并从标题填写开始重做。
```

### 保健食品类目属性规则

```text
保健食品类目属性按飞书字段填写生产企业名称、生产企业地址、净含量、产品标准代码、配料表和保健功能；保健功能必须精确匹配页面可见选项。保健食品不得套用医疗器械类目的 `型号规格=盒装` 基础信息规则。
```

### 主图与详情图规则

```text
上架商品时，主图生成节点产出的并且打了店铺水印的图片就是要上传的放在首位的主图，产品图片文件夹里的资质图片就是上架商品时详情页指定要上传的详情页图片。
```

### 图文信息规则

```text
图文信息只在“主图”模块上传；首位主图必须使用已通过图生图生成且已打店铺水印的 1:1 主图，再按固定顺序搭配 input/fixed-main-images/ 下的辅助图02-05；“主图3:4”和“白底图”板块不再属于项目发布流程，不上传、不点击智能裁剪、不清空、不读回、不阻塞；平台自动填充时保持平台状态即可。详情页必须先点击“从主图填入”，再上传产品文件夹里的资质图片作为详情页图片。
```

### 运费模板规则

```text
选择名称包含“延草运费”的运费模板。
```

### 规格模板规则

```text
标题含“久光小泽”则选择名称包含“久光小泽”的规格模板，否则选择名称包含“买二送一”的规格模板。
```

### 商品规格模块规则

```text
医疗器械和非处方药的商品规格名和值不允许编辑，完全依赖规格模板。保健食品选择“买二送一”模板并进入手动填写后，只允许在规格名精确为“规格”的规格组中，将唯一已填的规格值整体替换为飞书“规格”完整文本；不得解析或仅替换其中某个数字。
商品规格模块进入方式可能因抖店 UI 版本不同而变化；动作层必须以“商品规格”区域内真实可见的“规格模板”控件为准。若该控件已经可见，即使没有“切换手动填写”按钮，也应继续选择规格模板；若该控件不可见，归类为规格模板页面未就绪并保留断点重试。
保健食品规格替换必须精确定位 `商品规格 -> 规格 -> 已填规格值`。不得命中规格图片上传框、备注框、空白新增占位框、口味分类或件数。失焦稳定后必须精确读回飞书完整规格；读回不一致、规格组不唯一或已填规格值不唯一时立即停止，不进入价格库存。
```

### 保健食品包装标签规则

```text
保健食品必须在服务与履约读回通过后再上传包装标签图，上传目标限定为“包装标签图”字段，文件来源使用产品文件夹中的飞书资质图。包装标签图未读回成功时禁止进入发布前检查和最终发布。
```

### 医疗器械注册证规则

```text
医疗器械类目产品上架时，进入最后的“其他信息”模块检查“医疗器械注册证”字段：如果该字段已有填充内容，不做任何修改，继续后续上架；如果该字段为空，则直接上传飞书表格里的第一张资质图片，不走下拉框选择 SPU 信息方案。上传后必须确认该字段计数大于 0，仍为空则停止并保留断点。
```

### 服务与履约完成规则

```text
服务与履约相关必填项不得只按点击动作是否成功判定，必须以页面最终读回状态为准。发布前必须确认：发货模式=现货、现货发货时间=48小时、运费模板已选择有效的延草运费模板、商品状态=上架。任一字段未读回为完成时，停止并保留断点，不允许继续发布。
```

### 标品预填发布页规则

```text
标品页点击“发布商品”后，必须直接接管并复用该动作打开的 /ffa/g/create 发布页，不允许先关闭该页再重新访问 create URL。要求填写导购短标题但该页没有可操作的“导购短标题”字段时，视为标品预填发布页失效；必须关闭失效页并重新从标品页查询、点击“发布商品”获取新页，不允许通过持续刷新失效 create URL 恢复该字段。
```

### 店铺切换规则

```text
按店铺顺序执行发布：01 延草纲目大药房专营店、02 延草纲目药品专营店、03 延草纲目个护保健专营店、04 延草纲目康复理疗专营店、05 延草纲目医疗保健专营店、06 延草纲目理疗器械旗舰店、07 延草纲目健康护理专营店、08 延草纲目家庭护理专营店、09 延草纲目中医保健专营店、10 延草纲目养生器械专营店。医疗器械和保健食品依次上架 10 个店铺；非处方药只依次上架前 5 个店铺。
```

## 执行要求

1. 必须按店铺顺序执行，先完成当前店铺全部商品，再切换到下一个店铺。
2. 进入上架前，必须先检查右上角当前店铺是否与产品文件夹所在店铺一致。
3. 店铺不一致时，必须先走“切换组织/店铺”流程，切换成功后再继续。
4. 如果标品页仍在 loading、右上角店铺入口尚未渲染，不允许立刻判定店铺菜单缺失；必须等待店铺入口出现，必要时重进标品页后重试切店，最后一次仍失败才停止并截图。
5. 发布页“基础信息”模块是否激活，不能只依赖旧版字段；看到商品标题、导购短标题、商品类目、类目属性、品牌、医疗器械备案/注册号等当前页面字段时，也应判定基础信息模块可用。
6. 填写类目属性 `型号规格=盒装` 前，必须先在基础信息主表单内点击 `展开更多` 展开折叠的类目属性字段；如果展开后基础信息主表单没有 `型号规格` 输入框，不允许静默继续，也不允许把后续商品规格区的错误归因到基础信息填写成功。
7. 发布页因网络抖动出现内容消失、模块标签缺失、长时间 loading、系统繁忙或页面文本异常变短时，不允许继续点击和填表；必须先等待当前 `/ffa/g/create` 页恢复，必要时 reload 或重新进入当前 create URL，确认基础信息/图文信息/价格库存/服务与履约等模块和发布动作重新出现后再继续。
8. 发布页出现“数据异常请刷新重试”“网络异常”“系统繁忙”“请稍后重试”等可恢复平台异常时，即使页面上仍有模块标题或发布按钮，也不得判定页面可用；必须先刷新/重进 create URL，恢复后再继续动作。
9. 发布页仍显示模块标题或发布按钮但主表单处于 loading 时，仍属于页面未就绪，不允许继续填写。基础信息读回时，如果商品标题、导购短标题、型号规格等所有预期字段同时消失，属于发布页状态丢失，必须重新从标品页打开发布页并有限重试；如果仅个别字段缺失，则按真实字段校验失败停止。模块切换不能只依赖当前视窗内是否能看到字段；如果顶部 active tab 已经是目标模块，且主表单 DOM 中存在该模块字段，应视为模块已切换成功，再由后续字段级读回继续校验。
10. 模块内容可见性只能统计右侧主表单区域，不允许把左侧错误提示、顶部 tab 文案、填写助手、AI 助手或悬浮预览里的同名字段当成当前模块内容。
11. 顶部 active tab 已经精确等于目标模块时，不允许再报“expected 与 actual 相同”的模块切换失败；字段是否可操作必须交给该模块内部的字段级 ready/check 逻辑判断。
12. 每个模块进入下一模块前必须执行页面读回 gate，不能只依赖“已尝试填写字段”的内存记录；进入图文信息前必须读回基础信息里的商品标题、导购短标题、`型号规格=盒装` 均真实存在且没有红字错误；进入价格库存前必须再次读回基础信息 gate，防止基础信息未完成却继续填价格库存。跨模块 gate 只能读 DOM 状态，不允许为了校验主动切回旧模块页签。
13. 必须严格按照标题表格中的标题、导购短标题、品牌、SPU信息执行。
14. 商品详情图必须先点击 `从主图填入`，再上传飞书资质图；同一个商品的飞书资质图只允许上传一批，禁止因为页面计数延迟而整批重复上传。
15. 商品详情图最终数量必须刚好等于 `从主图填入后的数量 + 飞书资质图数量`；少于或多于预期都必须停止并保留断点。
16. 商品详情图是否完成必须以最终读回计数为准；如果上传动作的即时状态未确认，但最终计数已经刚好等于预期数量，视为详情图完成，不允许因瞬时状态误判失败。
17. 白底图/3:4 主图不属于项目发布流程；动作层不得上传、删除、清空、修复或读回这些槽位，规则层不得因为平台自动填充这些槽位而阻塞发布。
18. 不允许程序自行猜测、替换或模糊选相近标品。
19. 价格库存模块选择规格模板后，不修改规格名，不允许向空白规格值输入框键入内容。保健食品只允许把“规格”组唯一已填规格值整体替换为飞书完整规格文本；医疗器械和非处方药继续完全沿用模板规格值。规格模板是否生效仍以模板生成出价格库存/SKU 行为必要信号；价格库存已生成 4 行时，空白新增占位框不构成失败。
20. 页面保留的空白 `请输入规格值` 新增占位框不得填写，也不作为阻塞，且不得删除。`口味分类`和`件数`完全沿用模板，不修改。“买二送一”只允许按受控等价名 `买二送一`、`买2送1`、`2送1` 逐项搜索，不得选择“买三送二”等非等价模板。保健食品完整规格精确读回前禁止进入价格库存。
21. 商品规格模块未通过前，禁止进入 `价格与库存` 表格填写价格或库存；商品规格错误读取必须限定在右侧主表单的商品规格区域内，不能把左侧错误提示里的 `型号规格` 或 `该项为必填，请输入` 当成商品规格区错误；必须先让规格模板完成并且没有规格红字错误，再按飞书 `产品价格` 中正好 4 个从大到小的价格从上往下填入 4 行，库存统一填 2000。
22. 浏览器自动化必须使用 locator、角色、文本、label、DOM 邻近关系或 DOM 结构定位执行动作；不允许使用坐标点击、坐标移动或 `elementFromPoint` 作为上架动作兜底。
23. 主图生成节点产出的水印图作为首位主图。
24. 产品图片文件夹中的资质图作为详情图。
25. 同一商品流程只允许保留一个活跃 `/ffa/g/create` 发布页；重开发布页前必须关闭旧发布页。
26. 发布前检查未通过或详情页未完成时，必须停止并记录 checkpoint，不允许点击 `发布商品`；白底图/3:4 主图状态不参与此 gate。
27. 断点重跑时，如果当前产品已有发布成功截图或发布结果，必须跳过已发布商品，避免重复提交。
28. 浏览器动作和业务规则必须分开保存：页面点击、上传、截图属于动作层；是否算发布成功、是否能继续、是否必须停止属于规则层。
29. 医疗器械类目的“医疗器械注册证”字段必须按规则处理：已有填充物不覆盖，空白时上传飞书第一张资质图并校验成功。
30. 服务与履约模块必须按最终读回状态确认完成，不能把“点击过某个选项”当成字段完成。
31. 浏览器动作和业务规则必须分层：动作层默认只在当前模块执行本模块填写/修复；规则层决定能否进入下一模块。任何已离开模块后的普通回看校验必须只读，失败则停止并保留断点；白底图/3:4 主图不属于项目 gate，不做提交前清空动作。
32. 浏览器 CDP 调试端口探测和 Playwright 连接必须有硬超时。页面上下文丢失后，如果调试端口或连接处于半死状态，必须超时、重启可复用浏览器并进入有限发布重试，禁止无限等待。
33. 进入最终提交边界后，任何 `发布商品` 点击失败、页面上下文丢失、返回新建页、未拿到明确提交成功信号或平台提示瞬时异常，都属于最终发布状态不确定。动作层必须先在当前浏览器上下文内查找可用发布页、确认弹窗、提交成功页或新建页信号；仍无法确认时，不允许重新从 SPU 开页、重新填表、重新点击 `发布商品`。该单品必须记录为待核验失败并继续后续商品；只有后续实现稳定的商品管理列表查询并确认该商品确实不存在时，才允许自动补提。
34. 最终提交边界以前的页面就绪、规格模板、价格库存等失败必须记为 `not_checked` 安全未提交，可由精确断点续跑；只有已尝试点击 `发布商品` 且结果无法确认时才允许记为 `submit_accepted_unconfirmed`/人工复核，禁止把普通提交前失败误报为可能重复发布。
35. 一个发布子流程内出现多个失败目标时，任务终态、控制器状态和 Hermes 必须报告本轮最后实际阻塞项；不得继续用更早水印的历史错误覆盖当前失败。
36. 若真实 DOM 已显示 `商品规格`、`添加规格类型`、`规格预览`，但店铺完全没有 `规格模板` 控件，规则层必须归类为店铺规格模板未配置，而不是瞬态加载失败。动作层禁止手工编造规格名和值，禁止跳过该店铺继续后续目标，禁止反复刷新消耗恢复预算；当前目标记为 `not_checked` 并停止批次，由 Hermes 明确汇报需要用户在该店铺配置受控规格模板。配置完成后从原断点续跑。
37. 抖店登录失效不是项目可自驱修复的页面瞬态问题。动作层只允许使用固定自动化浏览器 profile 并做只读登录态预检；不得自动处理账号、密码、验证码或扫码登录。任一发布前检查发现 `doudian_login_required` 时，当前目标必须保留为 `not_checked`，停止批次并通过 Hermes 汇报人工登录需求；用户在同一自动化浏览器完成登录后，再从原断点续跑。

## 失败条件

- 店铺切换错误
- 标品查询值与表格不一致
- 主图或详情图识别错误
- 医疗器械类目“医疗器械注册证”字段为空且第一张飞书资质图上传失败
- 服务与履约字段最终读回缺失：发货模式、现货发货时间、运费模板、商品状态任一未完成
- 发布页关键字段未通过检查
- 发布页内容消失或模块标签缺失时未先恢复页面就继续点击/填表
- 同一商品存在多个活跃发布页
- 已发布商品被重复提交

## 人工检查点

1. 检查右上角当前店铺是否与产品所在店铺一致。
2. 检查发布页填入的标题、导购短标题、品牌、SPU是否与表格一致。
3. 检查主图是否为主图生成节点产出的水印图，详情图是否为资质图。
4. 检查当前商品是否已有发布成功记录，避免重复点击发布。
5. 医疗器械类目检查“其他信息/医疗器械注册证”是否已有内容；空白时应已上传飞书第一张资质图。
