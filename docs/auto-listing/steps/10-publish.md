# 步骤 10：商品上架

## 目标

按店铺顺序逐个产品图片文件夹执行上架，并保证整个发布流程严格按表格和固定规则执行。

## 动作脚本

- [publish.ts](C:/Users/ffc/.openclaw/workspace/douyin-title-generator/src/autolist/publish.ts)
- [publish-from-spu.ts](C:/Users/ffc/.openclaw/workspace/douyin-title-generator/src/business/publish-from-spu.ts)

## 输入

- 已完成商品信息回填的产品图片文件夹
- 已完成资质图补充的产品图片文件夹
- 店铺目录

## 输出

- 每个产品的发布结果

## 操作说明

### 发布执行原则

```text
一定要按照表格里的信息执行，包括标题、导购短标题、品牌信息、SPU信息等，不要盲目去猜测，按照表格里面的信息执行即可。
```

### 主图与详情图规则

```text
上架商品时，Dreamina CLI生成的并且打了店铺水印的图片就是要上传的放在首位的主图，产品图片文件夹里的资质图片就是上架商品时详情页指定要上传的详情页图片。
```

### 店铺切换规则

```text
一个店铺的产品上架完成后，必须切换到下一个店铺，再执行下一个店铺的上架。
```

## 执行要求

1. 必须按店铺顺序执行，先完成当前店铺全部商品，再切换到下一个店铺。
2. 进入上架前，必须先检查右上角当前店铺是否与产品文件夹所在店铺一致。
3. 店铺不一致时，必须先走“切换组织/店铺”流程，切换成功后再继续。
4. 必须严格按照标题表格中的标题、导购短标题、品牌、SPU信息执行。
5. 不允许程序自行猜测、替换或模糊选相近标品。
6. Dreamina 水印图作为首位主图。
7. 产品图片文件夹中的资质图作为详情图。

## 失败条件

- 店铺切换错误
- 标品查询值与表格不一致
- 主图或详情图识别错误
- 发布页关键字段未通过检查

## 人工检查点

1. 检查右上角当前店铺是否与产品所在店铺一致。
2. 检查发布页填入的标题、导购短标题、品牌、SPU是否与表格一致。
3. 检查主图是否为 Dreamina 水印图，详情图是否为资质图。
