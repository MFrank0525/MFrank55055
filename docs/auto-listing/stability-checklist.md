# 全链路稳定性整改清单

## 目标

把自动上架链路拆成可逐项整改、逐项验证的稳定性清单，后续每次只攻坚一个点。

## 当前结论

已基本稳定：

1. 店铺切换
2. 标题表格读取
3. 商品信息回填
4. 主图/详情图上传到发布前检查

仍需重点攻坚：

1. 豆包卖点结构化
2. DeepSeek 回复提取
3. Dreamina CLI 出图稳定性
4. 水印与产品文件夹落盘
5. 商品信息表主键匹配
6. 真正点击发布后的页面波动

## 整改项

### 1. 豆包卖点结构化

风险：

- 固定对话可能被历史内容污染。
- 第一个卖点后续仍要二次解析。

最稳定方案：

1. 豆包返回后立即固化：
   - `brand`
   - `userCognitionName`
   - `brandedGenericName`
   - `sellingPointSegments[8]`
2. 后续模块只消费结构化字段。
3. 拆不出关键字段就立即失败。

### 2. DeepSeek 回复提取

风险：

- 可能没进入指定会话。
- 可能抓到历史内容，提取不到 5 段。

最稳定方案：

1. 固定使用指定会话 URL。
2. 只截取本次 prompt 之后的最后一条回复块。
3. 结果必须正好 5 段，且每段为逗号关键词。
4. 不满足就失败，不写 Word。

### 3. Dreamina CLI 出图稳定性

风险：

- 单次不一定返回 4 张。
- Prompt 过重时可能直接失败。

最稳定方案：

1. 自动重复提交，累计到 4 张。
2. 明确区分：
   - 提交失败
   - `gen_status=fail`
   - 成功但不足 4 张
3. Dreamina 实际消费文案严格来自 Word 可采纳内容，不自行增删。
4. 每轮落盘：
   - `dreamina-prompt.txt`
   - `submit_id`
   - `query_result` 原始结果

### 4. 水印与产品文件夹落盘

风险：

- 半成品提前进入正式店铺目录。
- 产品文件夹外残留图片会干扰上架。

最稳定方案：

1. 所有图片先落任务临时目录。
2. 全部批次成功后，再统一落正式店铺目录。
3. 图片移入产品图片文件夹必须使用剪切粘贴式移动。
4. 产品图片文件夹外不允许残留同批次图片。
5. 测试前先清场。

### 5. 商品信息表匹配

风险：

- 名称变化可能匹配不到或匹配错行。
- 空值如果放行，会把错误带到发布页。

最稳定方案：

1. 保持空值即失败。
2. 新增产品主键映射：
   - `brandedGenericName -> 导购短标题 / 品牌 / SPU`
3. 运行时优先按主键映射读取。
4. 表格值与后台不一致时直接停止。

### 6. 发布页自动化

风险：

- 页面弹窗、模板选择、履约配置、点击发布都可能波动。

最稳定方案：

1. 分层测试：
   - `ready_before_publish`
   - 单产品真实发布
   - 跨店铺串行发布
2. 每个模块单独截图、单独校验。
3. 模块失败立即停止，不自动跳过。
4. 店铺切换后统一回固定标品管理 URL。

## 推荐执行顺序

1. 先稳生成侧：
   - 豆包卖点结构化
   - DeepSeek 回复提取
   - Dreamina CLI 出图
2. 再稳文件侧：
   - 水印落盘
   - 产品文件夹归档
   - 商品信息匹配
3. 最后稳发布侧：
   - `ready_before_publish`
   - 单产品真实发布
   - 跨店铺串行发布

## 执行原则

1. 每次只攻坚一个环节。
2. 已稳定步骤不重复改动。
3. 未触发步骤不提前联动。
4. 当前点失败，就先修当前点。
