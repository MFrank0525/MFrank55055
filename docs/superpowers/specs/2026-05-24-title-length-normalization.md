# 2026-05-24 标题长度归一化

## 根因

Hermes 启动的上架流程在第 15 个发布项停止，抖店发布页返回 `商品标题125/120 最长不能超过60个汉字（120个字符）`。原流程按“豆包生成标题不改动”保存标题工作簿，导致超长标题一直传递到发布节点，最终由平台拦截。

## 规则

1. 豆包生成标题 60 字以内，标题内容完全不改。
2. 豆包生成标题超过 60 字，只从标题前面删除多余字数，保留后 60 字。
3. 除前置截断外，不改写标题其他内容。

## 动作

1. 标题生成后、标题工作簿保存前，执行标题长度审查和归一化。
2. 标题工作簿中只保存已通过抖店 60 字上限规则的标题。
3. 已生成的历史标题工作簿可用同一规则临时修正，避免重新消耗豆包标题生成。
4. 发布流程只读取标题工作簿，不在发布页临时改标题。

## 验证

1. 单元测试覆盖 60 字以内原样保存。
2. 单元测试覆盖超过 60 字从前面删除多余字数。
3. 当前运行目录和店铺目录中的标题工作簿需确认标题长度不超过 60 字后再恢复 Hermes。
