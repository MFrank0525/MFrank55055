# 2026-05-24 标题缓存隔离与产品归属审计

## 根因

第三个产品 `医用芦荟凝胶 / 医用聚乙二醇护创敷料` 没有真正生成新标题，而是复用了第二个产品 `舒奈美医用医用重组Ⅲ型人源化胶原蛋白软膏` 的标题 CSV。

直接原因是标题动作层在整个 run 目录下查找 `doubao-title-output` 最新 CSV。多产品批次中，第二个产品留下的 CSV 对第三个产品仍然可见；第三个产品进入标题生成阶段时，动作层发现已有足量 CSV 后直接生成标题工作簿，没有再次调用豆包。

深层缺口是标题保存前缺少产品归属审计。标题表格虽然写进了第三个产品目录，但标题正文没有被校验是否属于当前飞书记录。

## 规则

1. 每个飞书产品必须拥有独立的标题生成缓存范围。
2. 标题 CSV 只允许在当前产品 task runtime 内复用，禁止跨产品复用同一 run 下的历史 CSV。
3. 医疗器械和非处方药标题保存前必须包含当前飞书记录的 `产品通用名称`。
4. 标题归属审计失败时必须在标题生成阶段失败，不能进入标题分发、信息回填、店铺分发或发布。
5. 保健食品标题没有固定通用名称后缀，暂不使用通用名称包含校验。

## 动作

1. `orchestrator` 调用标题生成时传入 `data/auto-listing/runs/<runId>/tasks/<taskId>` 作为标题运行目录。
2. `title-sheets` 只在该 task 目录下查找和生成 `doubao-title-output`。
3. `title-rules` 提供纯规则函数，负责判断标题是否属于当前产品。
4. `title-sheets` 在复用既有 CSV 和新生成 CSV 两个分支都调用规则函数，审计通过后才保存标题工作簿。
5. 第三个产品需要重新从白底图开始跑一遍，重新生成图片和标题，再重新发布；错误上架的旧商品不作为正确结果复用。

## 验证

1. 回归测试必须覆盖：当前产品通用名称匹配时通过，复用其他产品通用名称时失败。
2. TypeScript build 必须通过。
3. 规则完整性检查必须通过。
4. 重跑第三个产品时，运行目录中应出现新的 task 级 `doubao-title-output`，标题工作簿内容必须包含 `医用聚乙二醇护创敷料`。
