# 发布店铺切换韧性规格

日期：2026-05-23

## 根因

发布第 14 个商品时，页面截图显示当前右上角店铺已经是目标店铺 `延草纲目康复理疗专营店`。但右上店铺菜单处于加载/半透明状态，菜单里暂时没有可点击的 `切换组织/店铺` 入口。旧动作层把“当前店铺已匹配但切换入口暂不可用”误判为店铺切换失败，导致整批发布停止。

## 目标

1. 店铺上下文判断属于规则层，不藏在 Playwright 点击动作里。
2. 如果当前店铺已经匹配目标店铺，即使切换入口不可用，也视为上下文满足，继续发布。
3. 如果当前店铺不匹配且切换入口不可用，归类为可重试系统状态。
4. 只有明确不在目标店铺且多次无法打开/选择目标店铺时，才失败。

## 设计

规则层：`src/business/publish-from-spu/publish-rules.ts`

- 新增 `evaluateShopSwitchMenuState`：
  - `already_in_target_shop`：当前店铺匹配目标店铺。
  - `click_switch_entry`：当前店铺不匹配，且切换入口可见。
  - `retry_menu`：当前店铺不匹配，菜单打开但切换入口不可用。
  - `open_menu`：菜单未打开。
- 新增 `shop_switch_entry_unavailable` 错误分类，并纳入发布重试。

动作层：`src/business/publish-from-spu.ts`

- 在点击 `切换组织/店铺` 前读取菜单里的当前店铺。
- 调用规则层决策。
- 规则返回 `already_in_target_shop` 时直接关闭菜单并继续，不再强制点击切换入口。
- 规则返回 `retry_menu` 时刷新标品管理页并重试菜单。

## 验证

- 回归脚本覆盖“当前店铺已匹配但入口缺失”的场景。
- 回归脚本覆盖“入口缺失应归类为可重试系统状态”的场景。
