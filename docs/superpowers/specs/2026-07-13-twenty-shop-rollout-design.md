# 抖音全自动上架 20 店扩容设计

## 目标

把全自动上架项目从 10 家店铺扩展到固定顺序的 20 家店铺，同时保持每条飞书产品恰好 20 个发布目标：医疗器械和保健食品覆盖 20 家店、每店 1 个；非处方药只覆盖前 10 家店、每店 2 个。本次交付只跑通店铺访问与切换，不生成真实发布提交。

## 规则层设计

店铺目录模块只负责规范店铺身份、顺序及不可变副本。类目计划模块通过目录切片派生目标店铺：医疗器械和保健食品使用全部 20 家，非处方药使用前 10 家。每个计划显式记录 `targetsPerShop`，并由共享闭合校验确认 `shopCount × targetsPerShop = 20`、店铺编号唯一且均存在于规范目录。

主图水印、产品文件夹、标题分发、发布计划、完成度、进度和深度审计继续消费同一个类目计划。现有 `imagesPerShop` 若保留，应被明确视为“每店发布目标数”，避免另建含义重叠的字段。

## 动作层与编排设计

新增独立的只读店铺巡检能力，分为四个边界：

1. 巡检规则模块：判断期望店铺和实际店铺是否一致，验证证据序号连续、无重复、与规范目录完全相等。
2. 浏览器动作模块：打开抖店安全页面，调用现有 DOM 定位式店铺切换能力，切换后从菜单读取当前店铺名称并返回结果。
3. 巡检编排模块：严格按 01–20 顺序逐店执行，任一失败立即闭锁，不跳店、不把部分成功报告为完成。
4. CLI 入口：提供单一命令启动真实只读巡检并输出机器可读 JSON 摘要。

巡检代码不得导入发布任务编排、表单填写、最终提交或商品创建动作。巡检只允许导航、打开店铺菜单、切换店铺、读取 DOM、截图和写审计证据。最后一店验证完成后保持在当前安全页面，不恢复发布页，也不触发任何商品操作。

## 巡检证据

每次运行创建独立本地目录，保存：

- `shop-access-audit.json`：运行 ID、开始和结束时间、预期 20 店摘要、总体状态、失败位置。
- 每店结果：序号、`shopCode`、期望名称、读回名称、开始和结束时间、通过状态、失败分类。
- 失败时的 DOM 快照和截图；成功时不要求为每店生成截图，避免无价值产物膨胀。
- `sideEffects` 固定声明已执行的动作类别，并明确 `publishAttempted=false`、`formMutationAttempted=false`。

证据目录属于运行数据，不进入 Git。只有完整 20 条、严格有序、名称逐项匹配且无发布副作用的报告才能判为通过。

## 错误处理与恢复

- 登录失效：归类为 `login_required`，停止并保留证据。
- 店铺不存在：归类为 `shop_not_found`，停止；不得模糊匹配相似名称继续。
- 切换后读回不一致：归类为 `shop_context_mismatch`，有限重试后停止。
- 页面未就绪或平台异常：使用现有有限重试与安全导航策略；耗尽后停止。
- 证据写入失败：本次巡检失败，因为无法证明操作结果。
- 重新执行：创建新的运行目录，不覆盖旧证据；审计只接受显式指定或最新一次完整成功运行。

## 测试策略

按 TDD 实施：先写失败测试，再改生产代码。

1. 店铺目录测试精确断言 20 个编号和名称的顺序，且副本不可反向修改源数据。
2. 类目计划测试分别断言 20×1、20×1、10×2，并验证三类均恰好 20 个目标。
3. 主图和代表性模拟测试验证三类水印及目录分布的完整序列。
4. 巡检规则测试覆盖完整成功、乱序、重复、缺店、名称不匹配和副作用声明失败。
5. 模块边界测试证明只读巡检不依赖发布提交或表单动作。
6. CLI 契约测试验证失败闭锁、JSON 结构和非零退出码。
7. 进度与深度审计回归测试消除写死的旧 10/5 店假设。

## 审计与交付门禁

实现后先运行聚焦测试，再运行构建、规则检查、全部 doctor、飞书只读字段/API 检查和三类代表性模拟。真实抖店验证只执行 20 店只读巡检，禁止运行真实上架控制器。

随后执行两次独立审计：

1. 规则/矛盾审计：代码、类型、文档、示例、测试、状态分母和类目数量一致；搜索并人工判别所有旧 10/5 店表述。
2. 运行/副作用审计：巡检证据与 20 店目录逐项一致，身份与顺序无缺失、无重复，且未发生表单修改或发布尝试。

提交前检查 Git 状态，排除本地配置、凭据、浏览器资料、运行数据、截图和生成产物。只推送通过全部门禁的提交到当前 GitHub 主分支。

## 明确不变的范围

- 飞书字段契约不变。
- 每条产品仍生成 5 份提示词、20 张主图和 20 条标题。
- 标题、价格库存、资质、发布页面动作、恢复身份和清理规则保持不变。
- 本次不真实上架任何产品，不调用付费生图路径。
