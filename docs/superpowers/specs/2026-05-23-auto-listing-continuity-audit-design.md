# 抖音全自动上架连续性审计规格

日期：2026-05-23

## 背景

飞书表格中有多个产品时，自动上架流程必须按表格顺序依次处理全部待上架产品。此前出现过第一个产品完成后，清理动作删除了后续产品本地素材，导致后续产品无法继续的故障。虽然清理保护已经补上，但项目还缺少一个可重复运行的审计入口，无法在流程启动前或运行中快速判断“表格数据、已处理清单、本地素材、Hermes 状态”是否一致。

## 目标

1. 表格有 N 个产品时，系统能明确算出待处理产品数量，并发现缺失素材。
2. 已完成产品的清理动作不得删除未完成产品的白底图或资质图。
3. 规则和动作分离：连续性判断放在纯规则模块；文件读取、命令输出、退出码放在 CLI 动作层。
4. 审计命令必须只读，不修改运行数据，不触发真实上架。
5. 审计结果必须能被脚本和人类同时使用：默认文本输出，支持 `--json`。

## 非目标

1. 不在审计命令里调用飞书 API 重新下载素材。
2. 不在审计命令里启动 Hermes 或恢复上架流程。
3. 不重构主流程的大量业务步骤。

## 设计

规则层新增 `src/autolist/audit-rules.ts`：

- 输入：飞书记录、已处理图片、当前存在的白底图、当前存在的资质图、可选运行发现数量。
- 输出：`ok`、`errors`、`warnings`、`summary`。
- 规则：
  - 每条飞书记录必须至少有一张白底图。
  - 未处理产品的白底图必须存在。
  - 未处理产品的资质图如果在飞书数据中声明了本地文件，也必须存在。
  - 已处理清单中找不到飞书记录对应关系时给 warning，不阻断。
  - 如果当前运行发现的产品数少于规则计算出的待处理数，则报错。

动作层新增 `src/cli/audit-auto-listing.ts`：

- 读取自动上架 job 配置，默认 `input/auto-listing.job.json`。
- 读取飞书产品缓存、已处理清单、本地素材目录、最新 Hermes 状态文件。
- 调用纯规则模块输出文本或 JSON。
- 有 error 时退出码为 1，仅 warning 或全绿时退出码为 0。

测试：

- 扩展现有脚本测试纯规则：
  - 待处理产品素材完整时通过。
  - 未处理产品白底图缺失时报错。
  - 运行发现数量小于待处理数量时报错。
  - 清理选择仍保护后续产品素材。

文档：

- 在稳定性清单和飞书配置文档中加入 `npm run audit:auto-listing`。
