# 导购短标题预填页恢复设计

## 问题

标品页点击“发布商品”后首次打开的发布页包含“导购短标题”字段。当前流程随后关闭该浏览器上下文，只保存 create URL，再由新浏览器页面访问该 URL。抖店在这种二次访问中可能不再加载“导购短标题”，导致基础信息填写失败。

## 设计

“导购短标题”属于标品预填发布页完整性信号。商品要求填写导购短标题但发布页不存在该字段时，不刷新或重新访问当前 create URL，而是判定当前 SPU 预填发布页失效，关闭旧发布页并重新从标品页查询、点击“发布商品”获取新页面。

发布流程最多从标品页重开两次。新页面确认存在导购短标题字段后，才继续基础信息填写。超过次数仍缺失时停止并保留截图和断点。

普通网络异常、系统繁忙和页面 loading 仍使用现有发布页健康恢复逻辑；只有缺失预期导购短标题字段触发标品页重开。

## 代码边界

- 规则层 `src/business/publish-from-spu/publish-rules.ts` 判断预期基础字段是否要求重开标品预填页。
- 动作层 `src/business/publish-from-spu.ts` 读取导购短标题字段可见性，并在基础信息阶段触发已有的标品查询重开流程。
- `docs/auto-listing/steps/10-publish.md` 固化长期规则。
- `scripts/test-publish-create-page-readiness-rule.mjs` 和 `scripts/test-publish-module-sequence-rule.mjs` 提供回归覆盖。

## 验证

测试必须证明：

1. 要求导购短标题且字段缺失时，规则返回“从标品页重开”。
2. 基础信息阶段收到该结果时调用 `queryPlatformSpu()`，而不是刷新或重新访问失效 create URL。
3. 字段存在时继续原有填写流程。
4. 构建、规则检查、doctor 和模拟流程通过。
